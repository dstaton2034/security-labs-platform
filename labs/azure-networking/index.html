🏗️ Foundation
Hub-Spoke Topology
The recommended enterprise Azure network architecture — centralized security services in a hub, isolated workloads in spokes.
🏢

Hub VNet

Central networking services — Azure Firewall for egress control, VPN/ExpressRoute Gateway for cross-premises connectivity, Azure Bastion for secure VM access, shared DNS. All inter-spoke and outbound traffic routes through here.

FirewallGatewayBastionDNS
📦

Spoke VNets

Workload isolation — each spoke hosts a separate workload or environment (prod, dev, staging). Subnets divide tiers: web, app, data. Spokes peer to the hub only, not to each other (non-transitive by default).

IsolationMulti-tierSubscription-level
🔗

VNet Peering & UDRs

Peering connects hub to each spoke (low-latency Azure backbone). User Defined Routes (0.0.0.0/0 → Azure Firewall) force ALL spoke traffic through the hub firewall — no workload bypasses inspection.

Non-transitiveForced tunnelingGateway transit
🧩

Subnet Segmentation

Specialized subnets: AzureFirewallSubnet (requires /26), GatewaySubnet (VPN/ER), AzureBastionSubnet (requires /26+), workload subnets per tier. Each subnet can have its own NSG and route table.

AzureFirewallSubnetGatewaySubnetAzureBastionSubnet

🔒 Why It Matters

Hub-spoke is the Cloud Adoption Framework's recommended topology because it centralizes security control. Every packet between spokes or heading to the internet passes through the hub's firewall — giving you a single pane for network policy, logging, and threat detection. Without it, each workload manages its own egress and you lose visibility.

🛡️ Defense in Depth
7 Network Security Layers
From the Azure edge to the workload — each layer adds inspection, filtering, or isolation. An attacker must beat ALL of them.
1

Azure DDoS Protection

Always-on volumetric attack mitigation at the Azure edge. Standard plan adds adaptive tuning (learns your traffic baseline), real-time telemetry, attack analytics, and cost protection guarantee.

Edge LayerL3/L4
2

Azure Front Door + WAF

Global HTTP load balancing with integrated Web Application Firewall. OWASP Core Rule Set 3.2, custom rules, bot protection, rate limiting, geo-filtering. TLS terminated at edge POP closest to user.

Global EdgeL7
3

Azure Firewall (Premium)

Centralized stateful L3–L7 firewall in the hub VNet. Network rules, application rules (FQDN filtering), DNAT/SNAT, TLS inspection (decrypt/re-encrypt), IDPS with 67,000+ signatures, URL filtering, web categories, threat intel feed.

Hub VNetL3–L7
4

Application Gateway + WAF v2

Regional L7 load balancer with WAF. SSL offload, cookie-based affinity, URL path-based routing, auto-scaling. WAF v2 with per-site and per-URI policies for granular protection.

Spoke EntryL7
5

Network Security Groups (NSGs)

Stateful L3/L4 packet filtering at subnet or NIC level. 5-tuple rules (src IP, dst IP, src port, dst port, protocol). Priority-based (lowest number = first evaluated). Default: allow VNet, allow LB, deny all inbound.

Every SubnetL3/L4
6

Application Security Groups (ASGs)

Logical grouping of VMs by function (web servers, app servers, DB servers). Use in NSG rules instead of IP addresses — when you add a VM to the ASG, rules apply automatically. No rule updates needed.

LogicalZero-touch scaling
7

Private Endpoints

Private IP address in your VNet for PaaS services (Azure SQL, Storage, Key Vault). Traffic stays on Microsoft backbone — never traverses public internet. DNS resolution via Azure Private DNS zones.

🔮 The Full Picture
Where Microsoft Security Products Protect
The networking layers handle traffic control — but Microsoft's security stack lays ON TOP, adding detection, monitoring, and response at every level.
🌍
AZURE EDGE
DDoS + Front Door
🔭 Microsoft Sentinel
Ingests DDoS telemetry, Front Door WAF logs, and firewall diagnostics. Correlates network events with identity and endpoint signals. Runs analytics rules to detect network-based attacks (port scans, C2 callbacks, data exfiltration patterns).
☁️ Defender for Cloud — Network Recommendations
Continuously assesses your network posture: "Enable DDoS Standard on VNets with public IPs," "Restrict NSG rules allowing 0.0.0.0/0," "Enable WAF on Application Gateway." Secure Score reflects your network hygiene.
⬇️
🔥
HUB VNET
Firewall + Gateway
🔥 Azure Firewall IDPS + Threat Intel
67,000+ signatures detect known exploits, malware C2, and attack patterns in transit. Threat intelligence feed from Microsoft blocks traffic to/from known malicious IPs and domains in real-time.
🔐 Defender for Key Vault
Detects unusual access to Key Vault (secrets, certs, keys) from suspicious IPs or unexpected Azure services. Catches credential harvesting attempts. Alerts on access from Tor exit nodes or known malicious IPs.
⬇️
🖥️
SPOKE VNETS
Workload VMs
🛡️ Defender for Endpoint (MDE)
Kernel-level sensor on every VM. Detects post-breach activity: malware execution, credential dumping, lateral movement, persistence mechanisms. Network protection blocks connections to known malicious domains from the endpoint itself.
☁️ Defender for Servers
Vulnerability assessment (Qualys/MDVM), JIT VM access (opens NSG ports temporarily on-demand), adaptive application controls (allowlist running processes), file integrity monitoring. Plan 2 includes MDE.
🔍 Defender for DNS
Monitors ALL DNS queries from your VNets (using Azure DNS). Detects DNS tunneling (data exfiltration over DNS), connections to known C2 domains, domain generation algorithms (DGA), and crypto mining DNS patterns.
⬇️
🗄️
DATA LAYER
PaaS + Storage
🗄️ Defender for Azure SQL
Detects SQL injection attempts, anomalous database access patterns, brute-force login attempts, and access from unusual locations. Vulnerability assessment scans for misconfigurations (excessive permissions, unencrypted columns).
📦 Defender for Storage
Detects unusual access patterns to blobs and files: access from Tor, anonymous access enabled, mass download/deletion, malware uploads (hash reputation scanning), and sensitive data exposure.
🔏 Defender for Resource Manager
Monitors Azure control plane (ARM API). Detects suspicious management operations: NSG rules being deleted, firewall being disabled, public IPs being attached, role assignments from unusual locations.
⬇️
🔑
IDENTITY
Who accesses what
🔑 Microsoft Entra ID + Conditional Access
Every network connection starts with identity. Conditional Access policies enforce: MFA required from untrusted networks, block sign-ins from risky locations, require compliant devices for VPN access, restrict access to management portals by role.
🛡️ Defender for Identity (MDI)
Monitors on-prem Active Directory traffic. Detects lateral movement (pass-the-hash, pass-the-ticket, golden ticket), reconnaissance (LDAP enumeration, DNS recon), and privilege escalation — attacks that happen INSIDE your network.

🔮 How It All Connects — XDR + Sentinel

Every product above sends its signals to Microsoft Defender XDR (unified incident correlation) and Microsoft Sentinel (SIEM + SOAR). When Azure Firewall blocks a suspicious IP AND MDE detects malware on a VM AND Entra ID sees a risky sign-in — XDR automatically correlates these into a SINGLE incident with the full attack story. That's the power of a unified security stack on top of unified networking.

🔒 Why It Matters — Networking IS Security

The networking layers (DDoS, WAF, Firewall, NSGs, Private Endpoints) are your prevention — they block attacks before they reach workloads. The security products (Defender, Sentinel, Entra) are your detection and response — they catch what gets through and hunt for threats already inside. You need BOTH. A firewall without MDE can't detect a compromised insider. MDE without a firewall can't stop a DDoS. Together, they're defense in depth.
